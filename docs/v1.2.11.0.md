# v1.2.11.0

## 🐛 筛选导航 Bug 修复（v1.2.10.0 回归）

v1.2.10.0 的 OCR 视觉导航在部分分辨率/DPI 下会误判，导致筛选失败：

- **卡底检测误判**：高亮在可见区内移动时 OCR 文字不变，旧版连续 2 次不变即判"到底"放弃。新版加入高亮行下标追踪 + 64×64 像素差兜底，高亮在动就不判卡底
- **循环检测误判**：OCR 噪声导致页面指纹偶然重复即判"列表循环"。新版加 ≥3 步门槛 + 像素校验，画面有变化不判循环
- **回顶检测同类误判**：`_scroll_to_top` 文字不变时加像素差确认，高亮还在上移就不停
- **短目标 OCR 容错**：S1/S2/A 等 ≤3 字符目标被读成 SI/51/S 时精确匹配全 miss。新增常见易混字符模糊匹配（I↔1, 5↔S, Z↔2），S2 不会误匹配 S1

以上修复覆盖所有筛选场景（跑图选车/删车筛选，Steam/Xbox 共用 FilterNavMixin）。
