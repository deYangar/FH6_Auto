# v1.2.10.0

## 🔍 筛选导航重写：OCR 视觉导航替代固定按键（适配不同账号）

**背景**：旧版筛选（删车筛选 / 跑图选车）使用固定"按 N 下"导航。筛选列表内容因账号车辆拥有情况不同而变化（车型/稀有度条目可能缺失或增减），固定按键数在其他账号上会错位选错选项。

**新机制**：按文字目标导航，实时 OCR 可见列表 + 黄绿边框定位高亮行，自动适配任意账号的列表。

- 新增 `filter_nav.py`（FilterNavMixin）：`open_and_apply_filter(targets)` 全流程 = Y 打开 → X 重置残留勾选 → 逐个目标 OCR 定位 → 按键到位 → OCR 校验 → Enter 勾选 → ESC 关闭
- 高亮行检测：HSV 黄绿边框带（垂直膨胀粗定位 + 原始掩膜精修 + 内部黑底验证）；兜底策略为黑底行亮度检测
- 翻页搜索：半页步进（相邻页重叠，目标不会被跳过）+ 卡底/循环检测；目标在上方时自动回顶重搜
- 偏移校正：行差包含不可聚焦的黄色标题行，按键后 OCR 校验高亮文字，不符自动小范围上/下扫描校正
- 目标选项缺失（账号没有该类车）→ 报错并放弃该环节（不会带错误筛选继续跑，防卖错车）
- `ocr_onnx.py` 新增 `detect_lines_in_region()`：逐行 OCR 结果（文字 + 坐标，同行自动合并）

## ⚙️ 配置变更

方案新增两个字段（老配置启动时自动迁移，无需手动改）：

```json
{
  "sell_filter": ["重复项", "S2", "顶级超跑", "全轮驱动"],
  "race_filter": ["收藏", "复古拉力赛车", "传奇"]
}
```

- `sell_filter`：删车筛选目标（方案1 默认 重复项+S2+顶级超跑+全轮驱动；方案2 默认 S1+漂移赛车+后轮驱动+传奇）
- `race_filter`：跑图选车筛选目标（默认 收藏+复古拉力赛车+传奇 = Mad Mike 808 Wagon）
- 其他账号只需把选项文字改成自己列表里的实际文字即可，不用再数"按几下"
