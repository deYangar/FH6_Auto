# v1.2.10.0

## 🔍 筛选导航重写：OCR 视觉导航替代固定按键

旧版删车筛选/跑图选车用固定"按 N 下"导航，其他账号因车辆拥有情况不同会错位选错选项。新版按文字目标导航，自动适配任意账号：

- Y 打开筛选 → X 重置残留勾选 → 逐键 OCR 搜索，目标入屏瞬间直接 Enter（零过头）→ 复选框像素差验证 → 未命中自动回滚 + 点击兜底 → ESC 关闭
- 目标选项缺失（账号没有该类车）→ 报错放弃该环节，不带错误筛选继续跑（防卖错车）
- 配置变成文字目标列表，其他账号改 JSON 文字即可适配，不用再数"按几下"

## 👎 评价弹窗改为 OCR 点踩

赛事评价弹窗从图片匹配改为 OCR 识别，动作改为"下移一格 + Enter 点踩"。

## 🐛 Bug 修复（7 项）

- 修复二次 OCR 验证"按键没生效重按"分支必崩
- 修复 lambda 捕获 except 块变量（Python3 出块即删除）
- 修复 find_image_ultimate_safe 模板全失败时 UnboundLocalError
- 修复 config.json 损坏日志缺少异常原因
- 修复 recovery.py 裸 except
- 修复启动崩溃（log 节流变量未初始化）
- 修复截图缓存中毒（PrintWindow 失败后 GDI 对象被反复复用）

## ⚡ 性能优化

- OCR：rec 输入宽 320→240（耗时 -20~25%）；det 预处理缓冲复用；筛选面板 det 提速 ~44%
- 截图：DC/位图缓存复用（省 ~1ms/帧）；GDI 失败兜底不再崩 bot
- 模板匹配：灰度/边缘模板按 scale 惰性缓存
- 日志：100ms 批量节流渲染
- 筛选按键提速 ~2 倍 + 丢键容错
- 完赛检测间隔 1.0→0.5s
- 多线程匹配线程数 cpu//2，留更多核心给游戏

## 🔧 OCR 模型

换用 PP-OCRv6 small（det 9.4MB + rec 20.2MB，字典 18708 字符），移除 tiny（错误率过高）。

## ⚙️ 配置变更（自动迁移，无需手动改）

方案新增两个字段：

```json
{
  "sell_filter": ["重复项", "S2", "顶级超跑", "全轮驱动"],
  "race_filter": ["收藏", "复古拉力赛车", "传奇"]
}
```

- `sell_filter`：删车筛选目标（方案1 默认 重复项+S2+顶级超跑+全轮驱动；方案2 默认 S1+漂移赛车+后轮驱动+传奇）
- `race_filter`：跑图选车筛选目标（默认 收藏+复古拉力赛车+传奇 = Mad Mike 808 Wagon）
- 其他账号只需把选项文字改成自己列表里的实际文字即可

## 🔄 Xbox 版选车同步

Xbox 版跑图选车升级为与 Steam 版一致的 OCR 视觉导航 + OCR 上车检测。

## 🧹 代码清理

删除约 370 行死代码（旧选车流程遗留方法、未用导入、孤儿文件）；OCR 上车检测抽取为共享方法；调试截图统一；Xbox 版 OCR 区域对齐 Steam 版。
