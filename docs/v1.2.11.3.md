## v1.2.11.3 测试版

### 筛选导航修复
- **像素级卡底检测**：筛选面板到底后连续 3 次按↓像素无变化即判定到底，不再依赖 OCR 文本精确匹配，彻底解决 OCR 抖动导致的死循环
- **page_key 模糊比较**：允许 ≤1 行文本差异，OCR 单行抖动不破坏卡底检测
- **校正扫描收缩**：高亮验证失败后的来回扫描从 ↑8+↓16 缩至 ↑3+↓6，减少鬼畜
- **筛选面板下移 32px**：修正面板捕获区域底部截断，底部行（含勾选框□）完整识别
- **步数上限 80→62**：匹配满配列表 62 行，无需多余步数

### 超级抽奖优化
- **scale 缓存**：前 2 辆全量搜索 40 个缩放比（~5s/帧），第 3 辆起只搜缓存附近 5 个缩放比（<1s/帧），选车速度大幅提升
- **双层超时**：绝对超时 15s（防无目标时无限循环）+ 确认窗口 3s（从首次找到目标后计时）
- **页码记忆修正**：第 N 页找到车后下次直接从第 N 页开始搜索（修复原来 -1 导致永远少跳一页的 bug）
- **识别阈值恢复原版 0.85**：修复 UnsplashZ 调低到 0.82/0.80 导致错误车辆踩线通过的问题
- **strong_threshold 0.86→0.80**：高分候选第 1 帧即可通过兜底返回

### 删车逻辑修复
- **移除焦点兜底逻辑**：原逻辑在 removecarobject.png 未匹配时直接 Enter 打开焦点车卡验证"移除"按钮，会误删非目标车辆。恢复为主模板匹配 + 翻页的保守策略

### 线程安全与稳定性（来自 PR #22）
- 修复暂停/终止后重启任务的偶发闪退（心跳线程、后台输入、GDI 资源生命周期竞态）
- 新任务排队等待旧 runner 完全退出后再启动
- runner 使用启动时配置快照，不再跨线程读取 Tk 控件
- 单实例互斥，防止多实例同时控制游戏窗口
- Focus Hook 注入/卸载增加 5s 超时，已有模块复用避免重复注入

### 任务3 车辆识别改进（来自 PR #22）
- 改为视觉顺序优先（按列→行排序），不再选模板最高分车辆，避免跳过当前"全新"车
- 连续 2 帧空间一致性确认，过滤菜单动画/hover 瞬时误判

### 构建改进（来自 PR #22）
- build.bat 优先使用仓库 .venv，版本读取修复
- 新增 --smoke-test 打包验证参数
- Action 支持测试版 tag 前缀（test-v），测试版不会被正式版用户更新检测抓到
